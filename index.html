1. ¿Cuáles son las principales funciones de un kernel en la arquitectura de un sistema operativo y cómo interactúa con el hardware?

El kernel es como el "cerebro" del sistema operativo. Se encarga de gestionar los recursos del computador (como la memoria, la CPU y los dispositivos). Básicamente, es el intermediario entre las aplicaciones que usamos (como navegadores o juegos) y el hardware (como la memoria RAM, discos, etc.).

Gestiona procesos: Hace que los programas se ejecuten, asignando tiempo a la CPU.

Gestiona la memoria: Controla la memoria disponible, evitando que los programas se "pisen" entre sí.

Gestiona el hardware: Se comunica directamente con los dispositivos (impresoras, discos, etc.) usando controladores específicos.

Todo esto permite que las aplicaciones funcionen sin tener que entender cómo interactuar con el hardware directamente. ¡Es el puente entre los programas y el computador!

2. ¿Cómo se gestionan los procesos y cuál es la importancia de la multitarea en la eficiencia del sistema?

Los procesos son los programas que están en ejecución. El sistema operativo los organiza y gestiona, decidiendo qué proceso tiene acceso a la CPU y por cuánto tiempo. Esto se hace mediante un planificador de procesos.

La multitarea permite que varios procesos se ejecuten al mismo tiempo, aunque con una CPU única puede parecer que se hacen simultáneamente, ya que el sistema "salta" rápidamente de uno a otro.

¿Por qué es importante? La multitarea hace que el sistema sea mucho más eficiente. Mientras un proceso espera (por ejemplo, una respuesta de internet), otro puede seguir ejecutándose. ¡Esto hace que tu computadora trabaje de manera más fluida!

3. ¿Qué medidas de seguridad se pueden implementar a nivel de sistema operativo para proteger una red, especialmente en entornos empresariales?

A nivel de sistema operativo, hay varias estrategias para proteger una red:

  • Cortafuegos (Firewall): Filtra el tráfico de red, asegurando que solo las conexiones "buenas" puedan entrar o salir.
  • Contraseñas y autenticación: Usar contraseñas fuertes y herramientas de autenticación (como 2FA) para asegurarse de que solo personas autorizadas puedan acceder.
  • Actualizaciones constantes: Mantener todo actualizado ayuda a proteger el sistema de vulnerabilidades.
  • Seguridad en el acceso remoto: Para empresas, es clave usar redes privadas virtuales (VPN) para que los empleados se conecten de manera segura desde cualquier lugar.
  • Control de privilegios: Limitar quién tiene acceso a qué. Solo los usuarios necesarios deben tener permisos de administrador.

Estas medidas protegen tanto los dispositivos como la red, manteniéndola segura de ataques o accesos no autorizados.
